# 역별 피크 집중도 및 안정도 계산 설명

## 1. 분석 목적

지하철역마다 이용객이 특정 시간대에 몰리는 정도가 다르다.

예를 들어 출퇴근 중심의 역은 오전 또는 저녁 시간대에 이용객이 크게 몰릴 수
있다. 반면 상권이나 관광지 주변 역은 하루 동안 비교적 고르게 이용될 수 있다.

이러한 차이를 숫자로 비교하기 위해 아래 두 가지 지표를 계산했다.

- **피크 집중도(Peak Concentration)**: 가장 붐비는 시간대에 이용객이 얼마나
  집중되는지 나타낸다.
- **안정도(Temporal Stability)**: 하루 동안 시간대별 이용량이 얼마나 고르게
  분포하는지 나타낸다.

## 2. 사용 데이터

- 입력 파일: `final_merged_weekly_nonzero.csv`
- 출력 파일: `station_peak_concentration_stability.csv`
- 대상 기간: 2025년 1월 1일~12월 31일
- 대상 노선: 서울 지하철 1~8호선
- 분석 대상: 총 240개 역
- 이용량 기준: `이용객수_전체`

시간대는 아래와 같이 총 20개 구간으로 나뉜다.

```text
06시간대이전
06-07시간대
07-08시간대
...
23-24시간대
24시간대이후
```

승차와 하차는 구분하지 않고 모두 더했다.

같은 역명이 여러 호선에 존재하는 환승역도 하나의 역으로 합산했다.

```text
서울역 = 1호선 서울역 + 4호선 서울역
```

역명 변경으로 인해 같은 역이 분리되지 않도록 아래 이름도 통일했다.

```text
삼각지 → 삼각지(전쟁기념관)
당고개 → 불암산
```

## 3. 1년 합산 방식

이번 결과는 특정 날짜 하루의 이용량으로 계산한 값이 아니다.

2025년 1년 동안의 이용량을 역별·시간대별로 모두 더한 뒤 계산했다.

예를 들어 가상의 `A역`에 대해 1년 동안 `08-09시간대` 이용량을 모두 더하고,
같은 방식으로 나머지 19개 시간대의 이용량도 각각 더한다.

```text
A역의 1년 합산 시간대별 패턴

06시간대이전       100,000명
06-07시간대        200,000명
07-08시간대        500,000명
08-09시간대      1,000,000명  ← 가장 이용량이 많은 시간대
...
24시간대이후        50,000명
```

즉, 결과는 **2025년 전체를 기준으로 보았을 때 이 역의 대표적인 하루 시간대
패턴이 어떤 모양인지** 보여준다.

모든 시간대에 동일한 일수를 적용하여 일평균으로 바꾸더라도 피크 집중도와
안정도는 동일하다. 모든 시간대 값이 같은 비율로 나누어지기 때문이다.

## 4. 전체·평일·주말 구분

각 역마다 아래 세 가지 기준으로 별도 계산했다.

| `주말구분` | 의미 |
| --- | --- |
| `전체` | 1년간 평일과 주말 이용량을 모두 합산 |
| `평일` | 1년간 평일 이용량만 합산 |
| `주말` | 1년간 주말 이용량만 합산 |

따라서 한 역마다 결과가 3행씩 존재한다.

```text
가락시장,전체,...
가락시장,평일,...
가락시장,주말,...
```

총 240개 역을 계산했으므로 결과 파일은 헤더를 제외하고 720행이다.

## 5. 피크 집중도

### 5.1 계산식

```text
피크 집중도
= 하루 중 가장 이용량이 많은 시간대의 이용량 / 하루 전체 이용량
```

이번 분석에서는 하루 단위 원본 대신 1년 합산 패턴을 사용했으므로 실제
계산은 아래와 같다.

```text
피크 집중도
= 1년 합산 기준 가장 이용량이 많은 시간대의 이용량
   / 1년 합산 기준 전체 시간대 이용량
```

### 5.2 해석

피크 집중도가 높을수록 특정 시간대에 사람이 많이 몰리는 역이다.

```text
피크 집중도 = 0.17
```

위 값은 전체 이용량 중 약 17%가 가장 붐비는 하나의 시간대에 집중되어
있다는 뜻이다.

반대로 피크 집중도가 낮으면 이용객이 여러 시간대에 상대적으로 고르게
분산되어 있다는 뜻이다.

## 6. 안정도

### 6.1 계산식

```text
안정도
= 1 - (시간대별 이용량 표준편차 / 시간대별 이용량 평균)
```

괄호 안의 값은 변동계수(Coefficient of Variation)다.

### 6.2 해석

안정도가 높을수록 하루 동안 시간대별 이용량 차이가 작다.

즉, 특정 시간대에만 급격하게 몰리지 않고 비교적 고르게 이용되는 역이다.

안정도가 낮을수록 시간대별 차이가 크다. 출퇴근 시간처럼 특정 시간대에
이용객이 몰리는 역은 안정도가 낮게 나올 수 있다.

### 6.3 주의점

이 지표에서 말하는 안정도는 **계절별 또는 날짜별 이용량 변화가 적다**는
뜻이 아니다.

이번 안정도는 1년 합산 패턴을 기준으로, **하루 안에서 20개 시간대의
이용량이 얼마나 고르게 분포하는지**만 나타낸다.

또한 계산식의 특성상 시간대별 편차가 매우 크면 안정도가 음수가 될 수도
있다. 이번 결과에서는 모든 값이 양수로 나왔다.

## 7. 결과 파일 컬럼

| 컬럼 | 의미 |
| --- | --- |
| `역명` | 환승 노선을 합산한 역 이름 |
| `주말구분` | `전체`, `평일`, `주말` 중 하나 |
| `전체이용객수` | 해당 조건에서 1년 동안 합산한 이용객 수 |
| `피크시간대` | 이용량이 가장 많은 시간대 |
| `피크시간대_이용객수` | 피크 시간대의 1년 합산 이용객 수 |
| `피크집중도` | 피크 시간대 이용량이 전체 이용량에서 차지하는 비율 |
| `시간대별_평균` | 20개 시간대 이용량의 평균 |
| `시간대별_표준편차` | 20개 시간대 이용량의 표준편차 |
| `안정도` | 시간대별 이용량이 고르게 분포하는 정도 |

## 8. 결과 예시

`가산디지털단지`의 평일 결과는 아래와 같다.

```text
피크시간대: 08-09시간대
피크집중도: 0.185810
안정도:     0.017222
```

평일 전체 이용객 중 약 18.6%가 `08-09시간대`에 집중되어 있다.

안정도도 낮으므로, 하루 동안 이용량이 고르게 분포하기보다는 특정 시간대에
이용객이 크게 몰리는 출근 시간 중심 패턴으로 해석할 수 있다.

## 9. 활용 방법

두 지표를 함께 보면 역의 시간대 패턴을 간단하게 구분할 수 있다.

| 피크 집중도 | 안정도 | 해석 |
| --- | --- | --- |
| 높음 | 낮음 | 특정 시간대에 이용객이 뚜렷하게 몰리는 역 |
| 낮음 | 높음 | 하루 동안 비교적 고르게 이용되는 역 |
| 중간 | 중간 | 일부 피크가 있지만 지나치게 집중되지는 않는 역 |

현재 결과 파일에는 임의의 높음·중간·낮음 기준을 추가하지 않았다.

분류가 필요하다면 전체 역의 분포를 확인한 뒤 분위수 등을 기준으로 구간을
정하는 것이 적절하다.

## 10. 실행 방법

계산 스크립트와 입력 CSV를 같은 폴더에 두고 아래 명령을 실행한다.

```bash
python3 calculate_station_peak_metrics.py \
  final_merged_weekly_nonzero.csv \
  station_peak_concentration_stability.csv
```

추가 패키지는 필요하지 않다. Python 표준 라이브러리만 사용한다.
